Using Cascading Card Component

Some quick example text to build on the card title and make up the bulk of the card's content.

Code and Customization

To use the component simply wrap your card's content in the following:

<cascading-card>...</cascading-card>

At first, the component appears as a featureless surface. It can easily be transformed into a stylish (and customizable!) display tool. It's achieved by using its four attributes.

Attributes
src

What puts "cascading" into "cascading card" is an image. The attribute is used to specify the source of the picture - can be an URL or a relative path.
headerType

The attribute is used to decide the context an image has inside the component. By default, picture and card containing it have the same width, but setting headerType to wide or narrow changes that.
title

To easily set title for our card, we can add optional title attribute. The title passed in will be displayed as a <h4> stronged header right under the picture.

alignment

The final attribute is rather self-explanatory. It applies to card's content, and to be specific - everything passed in as <ng-content>, along with the title.

It accepts right and left arguments; by default it's set on center.

Code and Customization

To use the component simply wrap your card's content in the following:

<panel>...</panel>

At first, the component appears as a featureless surface. It can easily be transformed into a stylish (and customizable!) display tool. It's achieved by using its four attributes.

Attributes
header

Including this attribute with some text will create a <div> element in the upper part of the the very panel, just as in the example above.
footer

The attribute works symmetrically to header, except including it creates a footer in the bottom part of the panel.
color

To add consistency to our layout, we can specify color to be applied to header and footer sections of the panel
alignment

The final attribute is rather self-explanatory. It applies to panel's content, and to be specific - everything passed in as <ng-content>, along with the footer and header.

It accepts right and left as arguments; by default it's set on center.
